想用版本去规范一下自己写的程序

google 了一下软件版本号规范、风格、规则，也没找到什么好网页，所以还是觉得按自己的想法好了。

约定：

  • init版本为0.1.0
  • 每次小阶段开发就增加尾号，小阶段暂时为： bugfix、refactor
  • 增加一些重要的更新时，中间版本号为1，尾号重置为0
  • 看起来第一个0永远都达不到啊【碎碎念】
  • 在readme.md上增加一个changelog节，写明小阶段开发的change

  • 增加

  • 修改， 看起来这个打标签是比较慎重的行为，写错了只能删除。

  • 删除

  • 远端删除
